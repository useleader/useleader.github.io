关系数据库标准语言SQL

3.1 SQL概述

  • SQL Structured Query Language
    • 结构化查询语言，是关系数据库的标准语言
  • SQL是一个通用的、功能极强的关系数据库语言

3.1.1 SQL的起源和发展

3.1.2 SQL语言

  1. 综合统一
    • 数据定义DDL、数据操纵DML、数据控制DCL功能于一体，还有事务控制语言TCL
    • Data Definition Language: Create/Alter/Drop—schema定义
    • Data Manipulation Language: Insert/Select/Update/Delete — 数据查询及修改
    • Data Control Language: Grant/Revoke—权限修改
    • Transaction Control Language: Begin Transaction/Commit/Rollback—数据修改的原子性
    • 可以独立完成数据库生命周期中的全部活动
    • 可根据需要随时逐步修改模式，不影响运行
    • 数据操作统一
  2. 高度非过程化
  3. 面向集合的做做方法
  4. 以同一种语法结构提供多种使用方法
    • SQL独立语言
    • SQL嵌入式语言
    • 更多的采用数据库驱动程序API
  5. 语言简洁
    • 核心功能只用9个动词
      • Create、Alter、Drop
      • Insert、Delete、Update、Select
      • Grant、Revoke

3.1.3 SQL基本概念

  • 基本表
    • 本身独立存在的表
    • SQL中一个关系对应一个基本表
    • 一个基本表对应一个存储文件
    • 一个表可以有若干索引
  • 存储文件
    • 逻辑结构 schema 组成了关系数据库的模式
    • 物理结构
    • 文件路径和文件名可由用户配置或指定
  • 视图
    • 从一个或几个基本表通过查询而导出的表
    • 视图是虚表

3.2 学生-课程数据库

  • 学生-课程模式 S-T：
    • 学生表：Student(Sno, Sname, Ssex, Sage, Sdept)
    • 课程表：Course(Cno,Cname,Cpno,Ccredit)
    • 学生选课表：SC(Sno,Cno,Grade)
