数据库第11周学习日志

发表于 2023-11-16 更新于 2023-11-17 阅读次数：

0x00 前言

数据库完整性

0x01 实体完整性

1 实体完整性定义

关系模型的实体完整性

  • CREATE TABLE 用 PRIMARY KEY定义

单属性构成的主键有两种说明方法

  • 定义为列级约束条件
  • 定义为表级约束条件

对多个属性构成的主键只有一个说明方法

  • 定义为表级约束条件

2 实体完整性检查和违约处理

插入或对主码列进行更新操作时，关系数据库管理系统按照实体完整性规则自动进行检查

  • 检查主码值是否唯一
    • 全表扫描：耗时
    • RDBMS在主码上自动建立一个索引
  • 检查主码的各个属性是否为空

0x02 参照完整性

1 参照完整性定义

  • 用FOREIGN KEY定义外码
  • 用REFERENCES指明外码参照哪些表的主码

2 参照完整性检查和违约处理

  • 将两个表中的相应元组联系起来

  • 两个表的增删改都有可能破坏参照完整性，必须进行检查

  • | 被参照表（例如Student） | 参照表（例如SC） | 违约处理 |
    | —————————————- | ————————— | ———————————— |
    | 可能破坏参照完整性 | 插入元组 | 拒绝 |
    | 可能破坏参照完整性 | 修改外码值 | 拒绝 |
    | 删除元组 | 可能破坏参照完整性 | 拒绝/级连删除/设置为空值 |
    | 修改主码值 | 可能破坏参照完整性 | 拒绝/级连修改/设置为空值 |

  1. 拒绝执行 NO ACTION：默认策略
  2. 级联操作 CASCADE：
  3. 设置为空值 SET NULL：
  • 除了定义外码，还应该定义外码列是否允许空值

0x03 用户定义的完整性

  • 针对某一具体应用的数据必须满足的语义要求

1 属性上的约束条件

属性上约束条件的定义

  • 列值非空 NOT NULL
  • 列值唯一 UNIQUE
  • 检查列值是否满足一个条件表达式 CHECK

属性上的约束条件检查和违约处理

  • 约束条件RDBMS自动检查
  • 不满足则拒绝执行

2 元组上的约束条件

元组上约束条件的定义

  • CHECK短语定义元组上的约束条件，元组级的限制
  • 元组级限制可以设置不同属性之间的取值的相互约束条件

元组上的约束条件检查和违约处理

  • 约束条件RDBMS自动检查
  • 不满足则拒绝执行

0x04 完整性约束命名子句

1 完整性约束命名子句

CONSTRAINT <完整性约束条件名> <完整性约束条件>

2 修改表中的完整性限制

使用 ALTER TABLE语句修改表中的完整性限制

0x05 域中的完整性限制

0x06 断言

  • SQL中，使用CREATE ASSERTION 语句，通过声明性断言来指定更具一般性的约束
  • 可以定义涉及多个表的或聚集操作的比较复杂的完整性约束
  • 任何对断言所涉及的关系的操作都会触发DDBMS的检查，断言不为真值的操作会被拒绝。

1 创建断言的语句格式

CREATE ASSERTION <断言名>

2 删除断言的语句格式

DROP ASSERTION <断言名>

0x07 触发器

  • 触发器 Trigger 是用户定义在关系表上的一类由事件驱动的特殊过程
  • 触发器保存在数据库服务器中
  • 任何增删改操作均由服务器自动激活相应的触发器

1 定义触发器

CREATE TRIGGER

{BEFORE| AFTER} ON

REFERENCES NEW|OLD ROW AS

FOR EACH {ROW| STATEMENT}

[WHEN ]

  • 表的拥有者才可以在表上创建触发器
  • 触发器名
    • 可以包含模式名，也可以不包含模式名
    • 同一模式下，触发器名必须唯一
    • 触发器名和表名必须在统一模式下
  • 表名
    • 触发器只能定义在基本表上，不能定义在视图上
  • 触发事件
    • INSERT\DELETE\UPDATE
    • UPDATE OF {COLUMN}来限制具体某一列
  • 触发器类型
    • 行级 FOR EACH ROW
    • 语句级 FOR EACH STATEMENT
  • 触发条件：condition为真
  • 触发行动体：如果为行级触发器，可以使用new或old

2 激活触发器

  • 由触发事件激活
  • 可能定义多个触发器，顺序：
    1. 执行BEFORE触发器
    2. 激活触发器的SQL语句
    3. 执行AFTER触发器

3 删除触发器

  • DROP TRIGGER ON
