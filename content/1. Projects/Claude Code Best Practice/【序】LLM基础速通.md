## 1. Token机制：不是字，是子词

**什么是Token？**
Token是当前模型处理文本信息的基本单位，目前主流使用BPE（字节对编码）算法进行子词切分。

**纯字符 vs 纯词的困境：**
为什么不用纯字符或纯词？因为纯字符承载的语义太低，模型难以学习到长期依赖关系；而如果使用纯词，粒度又太粗，不仅参数量会巨大无比，还会遇到词汇表外（Out of Vocabulary）的新词问题。BPE子词切分正是在纯字符与纯词之间找到了绝佳的均衡点。

**英文 vs 中文（分词税）：**
目前主流分词器的词库主要是基于英文语料训练的，英文的高频词汇通常对应单个Token，平均约4个字符一个Token。而中文由于语料占比不足，遇到高频词块较少时，会退化为按单字或 UTF-8 的 Byte（1个汉字占3Byte）来切分。这就导致了同样信息量的中文内容需要消耗成倍的算力与成本，也就是所谓的**分词税**。

**多语言与多模态的”端到端（End-to-End）”处理：**
*   **多语言映射**：模型处理不同语言时，并不是先翻译成英文，而是将它们**统一映射到高维向量空间**中，在高维层次中捕捉本质含义。这种直接从原始输入映射到最终输出，无需人工介入（如强制翻译）的过程就是”端到端学习”。强制翻译反而会破坏特定语言独有的隐晦信息和局部流式结构造成信息损耗。
*   **多模态处理**：处理图像时，会将图片在物理上像切网格一样切分为一个个固定大小的图像块（Patch）。将这些物理切片转换为向量后，模型（如结合池化层或注意力机制）能在宏观上将信息结合起来，从而理解整体的特征（如颜色、形状等信息）。

## 2. 自回归生成：一步一步预测未来

**核心原理：**
自回归（Autoregressive）是指根据上一个Token的输出，将其加入到当前的输入中，去预测下一个Token，持续不断循环，直到输出最终的结束字符。
*数学表达*：`P(token_{t+1}) = P(token_{t+1} | token_{t}, token_{t-1}, ..., prompt)`

**选择策略（解码机制）：**
1.  **贪心策略（Greedy）**：每一步都直接选择当前预测概率最高的Token，确定性强，结果可重复。
2.  **采样策略（Sampling）**：按概率分布随机选择，随机性强，难以复现。
3.  **束搜索（Beam Search）**：一种启发式的图搜索算法。它不会只看眼前，而是同时保留当前概率总和最高的 $K$ 条路径，并基于这些路径各自往下预测，再从中择优录取，从而避免模型陷入局部最优的”死胡同”。

## 3. Context Window：上下文窗口

**什么是Context Window？**
上下文窗口是模型一次能处理的最大Token数，决定了模型真正能”记住”的范围。

**为什么有限制？（注意力机制带来的算力瓶颈）**
语言的含义高度依赖上下文，Transformer 的核心是自注意力机制（Self-Attention），这意味着序列中的每一个Token都必须与其他的每一个Token产生计算关系来决定权重。因此，计算量随着序列长度呈 $O(n^2)$ 呈指数级增长，这在物理硬件层面限制了窗口的无限扩大。

**超出窗口与提示词缓存（Prompt Caching）最佳实践：**
当对话超出窗口长度时，受限于滑动窗口机制，**最早期开头的信息会被截断遗忘**。由于人类对话习惯把重要前提放开头，这会导致模型忽略关键规则。
为了解决这个问题并优化算力，工程上的最佳实践是：
*   将核心规则（System Prompt）和大量静态模板放置在**信息流的最头部**。
*   不仅能强制锁定重要信息，还能极大提高**提示词缓存（Prompt Caching）**的命中率。将随时变化的用户提问放在末尾，一旦用户指令改变，头部的静态缓存依然有效。

## 4. Temperature / Top-p：控制随机性

两者都是用来在”采样策略”中控制模型输出随机性的参数。

**Temperature（温度）：**
Logits 是神经网络最后一层输出的**未经归一化的原始得分**。这些得分通过 Softmax 公式转化为总和为1的概率：`probability = softmax(logits / temperature)`。
*   **数学逻辑（纠偏）**：当 Temperature 小于 1 时，它会**放大**原本 Logits 之间的差距，使分布变得”尖锐”，输出更保守；当 Temperature 大于 1 时，它会**缩小**差距，使分布变得更”平缓”，输出更随机。
*   **极端情况**：温度不能在数学上直接除以 0，但在 API 设置中当温度趋近于 0 时，系统会直接将其底层策略切换为”永远选最高分”的贪心策略（ArgMax）。

**Top-p（核采样 / Nucleus Sampling）：**
它不固定候选Token的数量，而是动态拦截。
*   **运行机制**：将所有候选 Token 按概率从高到低排序，依次将它们的概率相加。当**累积概率之和**达到设定的 $p$ 阈值时，剩下的低概率内容全部砍掉。
*   **最终筛选**：截断后，剩下的高频词集合会根据它们自身的概率比例，重新进行一次随机抽样决定最终输出。
*   **极端情况**：当 Top-p = 1 时，代表累积到 100% 才截断，实际上等于禁用了 Top-p 机制。

## 5. Reasoning Model vs Fast Model

**什么是思考Token（Thinking Tokens）：**
传统的Fast Model（快模型）收到问题后直接生成回答；而 Reasoning Model（推理模型）会引入思维链（CoT）机制，先在内部（如 `<scratchpad>` 区域）进行逻辑推理，这部分消耗的就是思考 Token。

**为何思考过程”不可见”？**
在很多强大的推理模型中，思考 Token 是不向用户展示的。这不仅是为了界面清爽，更核心的商业原因是：**防止竞争对手抓取这些高质量的逻辑推演数据去蒸馏（训练）他们自己的模型**。如果在 API 中没看到它，不代表它没思考，而是厂商将其隐藏了。

**Effort 参数的作用：**
API 中设定的 `effort`（如 low, medium, high）对应的正是模型被允许消耗的**思考 Token 预算上限（推理强度）**。分配的上限越高，模型在给出最终答案前能进行的内部试错和推演就越深。
