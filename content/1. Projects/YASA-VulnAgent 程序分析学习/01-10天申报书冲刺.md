# 01-10天申报书冲刺

## 优先级

申报书成稿 > 项目链路讲清楚 > 程序分析细节深挖。

## 每日节奏

| 时间块 | 内容 |
|---|---|
| 30-45 分钟 | 复习前一天术语和项目链路 |
| 1.5-2 小时 | 阅读当天材料 |
| 2-3 小时 | 写申报书对应章节 |
| 30 分钟 | 晚间验收和第二天准备 |

## 10 天计划

| 天 | 快速学习 | 写作产出 | 验收检查 |
|---:|---|---|---|
| D1 | 通读 `requirements.md`、`research/README.md`、`CLAUDE.md`，画出总链路 | 项目定位、一句话摘要、申报书目录映射 | 能讲清“为什么不是纯 LLM，也不是传统 SAST” |
| D2 | 学传统 SAST、纯 LLM、混合方案差异，读相关工作综述 | 背景意义、国内外现状、技术空白 | 能说清 IRIS、SAILOR、VulWeaver 与本项目区别 |
| D3 | 学 YASA、UAST、YASA-Engine、YASA-MCP 边界 | 技术基础、现有工作基础、可行性素材 | 能解释 UAST 为什么是项目核心而不是装饰概念 |
| D4 | 学污点分析五件套：source、sink、sanitizer、propagator、filter | 关键技术问题初稿 | 能手工拆一个 SQLi source-to-sink 路径 |
| D5 | 学 LangGraph、Supervisor、Skill、MCP tool、PEV | 技术路线与研究内容初稿 | 能画出四个 Skill 的输入输出链路 |
| D6 | 学 PoC 验证、Docker 沙箱、CVE、CVSS、baseline 指标 | 阶段目标、成果形式、评价指标初稿 | 能说明如何验证“发现的漏洞是真的” |
| D7 | 聚焦创新点：UAST、grounding、PEV、端到端验证 | 创新点与先进性初稿 | 每个创新点都能对应技术机制和验证方式 |
| D8 | 串联所有章节，修逻辑断裂 | 完整申报书草稿 v1 | 读一遍能形成统一故事线 |
| D9 | 查风险：YASA-MCP API 边界、时间线、评估可行性 | 风险控制、实施进度、可行性修订 | 没有把推断 API 写成已确认事实 |
| D10 | 语言压缩、格式整理、附件和素材索引 | 可提交版申报书 | 章节完整、术语一致、目标不过度承诺 |
