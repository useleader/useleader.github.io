# 02-程序表示与UAST

## 学习目标

理解 AST、CFG、DFG、PDG、SSA、CPG、UAST 的区别，并能说明 YASA 为什么选择 UAST 支撑多语言静态分析。

## 核心概念

| 概念 | 一句话解释 | 在 YASA-VulnAgent 中的作用 |
|---|---|---|
| AST | 源码语法结构树 | 表示代码长什么样 |
| CFG | 控制流图 | 表示程序可能怎么走 |
| DFG | 数据流图 | 表示值从哪里来、流向哪里 |
| PDG | 程序依赖图 | 同时表达控制依赖和数据依赖 |
| SSA | 静态单赋值形式 | 让赋值关系更清楚 |
| CPG | 代码属性图 | Joern 等工具聚合 AST/CFG/DFG 的表示 |
| UAST | 统一抽象语法树 | 让多语言代码进入统一分析框架 |

## 本周练习

使用 `cases/SQLi贯穿案例.md` 的代码片段，分别画出简化 AST、CFG、DFG，并说明哪些信息适合放在 UAST 中，哪些信息需要后续数据流分析得到。

## 项目关联

申报书中描述 YASA 时，重点强调 UAST 是跨语言统一表示层；不要把 UAST 写成自动发现漏洞的完整答案。
