# 03-指针别名与数据流分析

## 学习目标

理解 points-to、alias、函数调用、对象字段和敏感性维度如何影响数据流与污点传播。

## 核心概念

| 概念 | 一句话解释 | 为什么影响漏洞分析 |
|---|---|---|
| points-to | 变量或引用可能指向哪些抽象对象 | 决定对象字段上的污点是否能被追踪 |
| alias | 两个变量可能引用同一对象 | 一次写入可能污染另一个变量读取 |
| flow-sensitive | 区分语句顺序 | 重新赋值或净化后结果不同 |
| context-sensitive | 区分调用上下文 | 同一函数被不同 caller 调用时结果不同 |
| field-sensitive | 区分对象字段 | 避免整个对象被粗暴污染 |
| path-sensitive | 区分分支条件 | 避免互斥路径造成误报 |

## 本周练习

改写 SQLi 案例，让用户输入先写入对象字段，再传入 SQL 拼接函数。解释 alias 和 field-sensitive 分析如何影响结论。

## 项目关联

YASA 的 point-to analyzer 是污点分析前置基础。申报书中可以把它作为降低误报、支持跨函数传播的关键技术基础。
