# 04-污点分析

## 学习目标

掌握 source、sink、sanitizer、propagator、filter、taint path，并能手工判断一条路径的可疑性和误报风险。

## 五件套

| 组件 | 含义 | 示例 |
|---|---|---|
| Source | 不可信输入来源 | HTTP 参数、请求体、cookie、上传文件 |
| Sink | 危险操作位置 | SQL 执行、命令执行、HTTP client、反序列化 |
| Sanitizer | 净化或约束风险的操作 | 参数化查询、HTML escape、allowlist |
| Propagator | 传播污点的操作 | 赋值、拼接、函数传参、返回值 |
| Filter | 过滤路径或规则 | 只保留某类 CWE、排除已净化路径 |

## 三类优先漏洞

| 漏洞 | CWE | 典型路径 |
|---|---|---|
| SQL 注入 | CWE-89 | HTTP 参数 -> SQL 字符串 -> DB execute |
| SSRF | CWE-918 | URL 参数 -> HTTP client 请求 |
| 命令注入 | CWE-78 | HTTP 参数或文件名 -> shell command |
