# 05-查询语言与漏洞规则

## 学习目标

理解声明式查询如何表达漏洞模式，并能写出 SQLi、SSRF、命令注入的伪 CodeQL/UQL 查询。

## 关键概念

| 概念 | 解释 | 项目作用 |
|---|---|---|
| 声明式查询 | 描述要找什么，而不是手写遍历过程 | 表达 checker |
| Predicate | 逻辑谓词 | 封装 source、sink、sanitizer 判断 |
| Recursion | 递归关系 | 表达跨函数路径 |
| Path query | 路径查询 | 输出 source-to-sink 证据 |
| Datalog | 事实和规则驱动的关系推导语言 | 理解数据流求解思想 |

## 本周练习

在 `outputs/漏洞伪查询集.md` 中写出三类漏洞的伪查询，并为每条查询说明可能误报来源。

## 项目关联

如果 YASA-UQL 或 MCP 查询能力可用，Skill-1 可以优先调用结构化查询；如果不可用，Phase 1 需要用已确认后端或替代工具验证同样的 source-to-sink 思路。
