# 命令注入污点路径案例

## 示例代码

```python
import os

def ping_host(request):
    host = request.args["host"]
    command = "ping -c 1 " + host
    return os.system(command)
```

## 污点标注

| 元素 | 代码 | 说明 |
|---|---|---|
| Source | `request.args["host"]` | 用户可控 host 参数 |
| Propagator | `host = request.args["host"]` | 赋值传播 |
| Propagator | `command = "ping -c 1 " + host` | 拼接进入 shell command |
| Sink | `os.system(command)` | shell 执行 |
| Sanitizer | 无 | 没有参数数组调用、allowlist 或 shell escaping |

## Source-to-Sink 路径

```text
request.args["host"]
  -> host
  -> command
  -> os.system(command)
```

## 误报检查

1. 如果 host 经过严格 IP/域名 allowlist 校验，风险需要重新判断。
2. 如果改为 `subprocess.run(["ping", "-c", "1", host], shell=False)`，风险通常低于 shell 拼接。
3. 如果执行环境权限极低，影响范围下降，但漏洞模式仍成立。

## 伪查询目标

查找用户输入经字符串拼接流向 shell 执行 API，且没有有效 allowlist 或参数数组调用。
