# 漏洞伪查询集

## SQL 注入

```ql
from Source src, Sink sink, TaintPath path
where
  src.isHttpInput() and
  sink.isSqlExecution() and
  path.flowsTo(src, sink) and
  not path.hasSanitizer("parameterized_query")
select sink, path, "Potential SQL injection"
```

误报来源：

1. `sink.isSqlExecution()` 把安全 ORM wrapper 误识别为原始 SQL 执行。
2. sanitizer 被封装在项目内部 helper 中，规则未识别。
3. 拼接 SQL 只用于日志或测试路径，没有真实执行。

## SSRF

```ql
from Source src, Sink sink, TaintPath path
where
  src.isHttpInput() and
  src.mayControlUrl() and
  sink.isServerSideHttpRequest() and
  path.flowsTo(src, sink) and
  not path.hasSanitizer("url_allowlist") and
  not path.hasSanitizer("private_ip_block")
select sink, path, "Potential SSRF"
```

误报来源：

1. allowlist 逻辑存在但规则没有建模。
2. HTTP client 被配置为只能访问固定代理。
3. 输入经过业务枚举映射，不直接控制 URL。

## 命令注入

```ql
from Source src, Sink sink, TaintPath path
where
  src.isHttpInput() and
  sink.isShellExecution() and
  path.flowsTo(src, sink) and
  path.hasStringConcatenation() and
  not path.hasSanitizer("strict_allowlist") and
  not sink.usesArgumentArrayWithoutShell()
select sink, path, "Potential command injection"
```

误报来源：

1. 输入被严格限制为 IP 或域名格式。
2. 执行 API 名称相似但不调用 shell。
3. wrapper 内部已经使用参数数组且 `shell=False`。
