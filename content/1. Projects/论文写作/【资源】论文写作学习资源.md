## 安全顶会投稿指南

### 官方资源

- [USENIX Security '26 Call for Papers](https://www.usenix.org/conference/usenixsecurity26/call-for-papers) - USENIX Security 2026征稿启事
- [IEEE S&P 2026 Call for Papers](https://www.ieee-security.org/TC/SP2026/cfpapers.html) - IEEE Symposium on Security and Privacy 2026征稿启事
- [ACM CCS 2026 Call for Papers](https://www.sigsac.org/ccs/CCS2026/call-for/call-for-papers.html) - ACM Computer and Communications Security 2026征稿启事
- [USENIX Security '25 Submission Policies](https://www.usenix.net/conference/usenixsecurity25/submission-policies-and-instructions) - 提交格式要求(双栏，10pt Times Roman)

### 关键格式要求

**USENIX Security:**
- 页面格式：US letter，双栏，10pt Times Roman，12pt leading
- 文本区块：7" x 9"
- 必须使用USENIX官方模板和样式文件
- 匿名审稿：标题页不得包含作者姓名或单位
- 每位作者每周期最多提交7篇论文

**IEEE S&P:**
- 两轮提交周期
- 必须先注册摘要(在提交截止日期前一周)
- 理论论文必须证明对实践的相关性
- rebuttal阶段(交互式)：8月中旬至8月底

**ACM CCS:**
- 双栏ACM格式 "sigconf"
- 不超过12页(不含参考文献、appendix和supplementary)
- 必须匿名化
- 需引用第三人称表述自己的前期工作

### 安全会议风格差异

| Conference | 偏好类型 | 特点 |
|------------|----------|------|
| IEEE S&P | 系统性贡献、长期影响 | 1980年起源于Oakland |
| USENIX Security | 实战型攻击、大规模实验 | 双重提交周期 |
| CCS | 理论与实践平衡 | ACM系列 |
| NDSS | 大规模验证、可复现性 | 网络安全优先 |

## 论文模板/格式指南

### LaTeX模板

- [USENIX Security LaTeX Template](https://www.usenix.org/conferences/usenixsecurity25/submission-policies-and-instructions) - USENIX官方模板
- [ACM Proceedings Template](https://www.acm.org/publications/proceedings-template) - ACM官方模板(for NeurIPS, ICLR, CCS, FSE, ASE)
- [ICML 2026 LaTeX Template](https://media.icml.cc/Conferences/ICML2026/Styles/example_paper.pdf) - ICML官方模板示例
- [IEEE Templates](https://www.ieee.org/publications/standards/publications/templates.html) - IEEE官方LaTeX和Word模板

### 格式要点

**NeurIPS/ICLR/ICML:**
- 使用对应会议的style文件
- natbib引用格式
- 图表使用graphicx指定宽度
- 表格建议使用booktabs

**USENIX Security:**
- Times Roman 10pt, single-spaced
- 双栏US letter
- 严格遵守页面限制

**ACM系列(FSE/ASE/CCS):**
- 使用ACM Primary Article Template
- 双栏sigconf格式
- 参考文献不计入页面限制
