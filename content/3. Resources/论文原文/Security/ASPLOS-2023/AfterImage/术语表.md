# AfterImage: 通过硬件预取器泄露控制流信息

## 技术术语

| 英文术语 | 中文术语 | 解释说明 |
| --- | --- | --- |
| AfterImage | AfterImage | 一种利用Intel IP-stride预取器的新型侧信道攻击，可跨代码区域、进程和用户-内核边界泄露控制流信息 |
| IP-stride Prefetcher | IP步幅预取器 | Intel处理器中基于指令指针的步幅预取器，通过学习同一IP的负载地址之间的规律性来预取数据 |
| Hardware Prefetcher | 硬件预取器 | 位于CPU和内存子系统之间的硬件组件，通过预测和预加载数据来隐藏DRAM访问延迟 |
| Side-channel Attack | 侧信道攻击 | 利用系统实现中的物理效应（如时序、功耗、电磁辐射）来获取敏感信息的攻击方法 |
| Cache Timing Attack | 缓存时序攻击 | 利用缓存访问时间的差异来推断敏感信息的攻击技术 |
| Control Flow | 控制流 | 程序执行过程中指令的执行顺序 |
| Prime+Probe | Prime+Probe | 一种缓存侧信道技术，攻击者首先填充缓存集合，然后探测哪些被受害者访问 |
| Flush+Reload | Flush+Reload | 一种缓存侧信道技术，利用共享内存，攻击者先刷新共享内存，然后检测受害者访问后的加载时间 |
| Microarchitecture | 微架构 | 处理器的硬件结构和组织方式，如Haswell、Coffee Lake等 |
| Spectre | Spectre | 一种利用处理器推测执行机制的侧信道攻击变体 |
| LRU | 最近最少使用 | 一种缓存替换策略，优先替换最长时间未被访问的数据 |
| Confidence | 置信度 | IP-stride预取器中的参数，决定是否触发预取 |
| Stride | 步幅 | 连续内存访问地址之间的间距 |
| MES | 最小驱逐集 | 能够完全驱逐缓存集合中所有数据的最小地址集合 |
| LLC | 最后级缓存 | 处理器缓存层次结构中最外层的缓存，被所有核心共享 |
| Context Switch | 上下文切换 | 操作系统在不同进程或线程之间切换时保存和恢复CPU状态的过程 |
| TLB | 翻译后备缓冲区 | 缓存虚拟地址到物理地址转换的硬件缓存 |
| Covert Channel | 隐蔽信道 | 违反安全策略的信息传输通道 |
| Speculative Execution | 推测执行 | 处理器在确认指令结果前提前执行可能需要的指令 |
| Branch Predictor | 分支预测器 | 预测条件分支跳转方向的硬件组件 |
