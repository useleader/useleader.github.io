# SSPO 与其他方法对比

## 方法全景图

```
LLM Alignment Methods
├── RLHF (PPO-based)
│   ├── InstructGPT (2022)
│   └── Various improvements
├── Direct Preference Optimization
│   ├── DPO (2023) - 开创者
│   ├── SimPO (2024) - 去除参考模型
│   ├── ORPO (2024) - 单阶段训练
│   └── KTO (2024) - 行为经济学
└── Semi-Supervised
    └── SSPO (2026) - 半监督学习
```

## 核心特性对比

| 方法 | 参考模型 | 训练阶段 | 数据格式 | 关键创新 |
|------|----------|----------|----------|----------|
| RLHF | 是 | 3 | Pair | 强化学习框架 |
| DPO | 是 | 2 | Pair | 直接优化 |
| SimPO | 否 | 1 | Pair | 去除参考模型 |
| ORPO | 否 | 1 | Pair | 单阶段+赔率比 |
| KTO | 否 | 1 | Single | 损失厌恶 |
| **SSPO** | **否** | **1** | **Pair+Single** | **半监督** |

## DPO vs SSPO

### DPO的局限性

1. **数据效率低**：需要大量成对偏好数据
2. **标注成本高**：每条数据需要人类比较
3. **忽视无标注数据**：SFT数据未被利用

### SSPO的改进

| 方面 | DPO | SSPO |
|------|-----|------|
| 标注数据需求 | O(N) | O(0.1N) |
| 无标注数据利用 | ✗ | ✓ |
| 伪标签 | ✗ | ✓ |
| 理论保证 | 隐式 | 显式阈值 |

### 核心区别

```
DPO: 只能利用标注数据
SSPO: 同时利用标注 + 无标注数据（通过伪标签）
```

## SimPO vs SSPO

### SimPO的局限性

1. 仍然依赖大量标注数据
2. 没有利用无标注SFT数据
3. 目标margin需要人工调参

### SSPO的优势

- 理论上最优的阈值选择
- 自动从数据中发现隐式偏好
- 更好的数据效率

## ORPO vs SSPO

### 相似点

- 都追求单阶段训练
- 都不需要参考模型
- 都优化隐式奖励

### 不同点

| 方面 | ORPO | SSPO |
|------|------|------|
| 数据利用 | 仅标注数据 | 标注+无标注 |
| 训练目标 | SFT+赔率比 | 二分类+伪标签 |
| 理论框架 | 赔率比 | 最优阈值 |

## KTO vs SSPO

### KTO的独特性

- 使用单样本格式（不需要pair）
- 基于前景理论
- 对称损失

### SSPO的独特性

- 半监督框架
- 伪标签技术
- 更强的数据效率

## 数据效率对比

基于论文实验（UltraFeedback, Mistral-7B）：

```
性能对比 (相对于DPO 10%):
├── DPO 1%:  baseline (差)
├── DPO 10%: 100% (参考)
├── SimPO 1%: ~85%
├── ORPO 1%: ~88%
├── KTO 1%: ~90%
└── SSPO 1%: ~95-100% ✓
```

## 理论框架对比

### DPO：信息论视角

- 最小化策略与参考策略的KL散度
- 隐式奖励函数

### KTO：行为经济学视角

- 损失厌恶
- 前景理论价值函数

### SSPO：半监督学习视角

- 伪标签蒸馏
- 最优阈值理论

## 何时使用哪种方法？

| 场景 | 推荐方法 |
|------|----------|
| 数据充足，追求稳定 | DPO |
| 想要简单实现 | SimPO |
| 想单阶段训练 | ORPO |
| 有单样本数据 | KTO |
| **数据稀缺，想最大化效率** | **SSPO** |

## SSPO的独特价值

1. **首个半监督偏好优化**：填补了这一空白
2. **理论保证**：最优阈值的存在性证明
3. **惊人的数据效率**：1%数据媲美10%数据
4. **框架通用性**：可扩展到其他模态
