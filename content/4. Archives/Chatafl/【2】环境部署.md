### 1.3. Running Experiments

Utilize theô `run.sh`ô script to run experiments. The command is as follows:

```shell
 ./run.sh <container_number> <fuzzed_time> <subjects> <fuzzers>
```

Whereô `container_number`ô specifies how many containers are created to run a single fuzzer on a particular subject (each container runs one fuzzer on one subject).ô `fuzzed_time`ô indicates the fuzzing time in minutes.ô `subjects`ô is the list of subjects under test, andô `fuzzers`ô is the list of fuzzers that are utilized to fuzz subjects. For example, the command (`run.sh 1 5 pure-ftpd chatafl`) would create 1 container for the fuzzer ChatAFL to fuzz the subject pure-ftpd for 5 minutes. In a short cut, one can execute all fuzzers and all subjects by using the writingô `all`ô in place of the subject and fuzzer list.

When the script completes, in theô `benchmark`ô directory a folderô `result-<name of subject>`ô will be created, containing fuzzing results for each run.

### 1.4. Analyzing Results

Theô `analyze.sh`ô script is used to analyze data and construct plots illustrating the average code and state coverage over time for fuzzers on each subject. The script is executed using the following command:

```shell
./analyze.sh <subjects> <fuzzed_time> 
```

The script takes in 2 arguments -ô `subjects`ô is the list of subjects under test andô `fuzzed_time`ô is the duration of the run to be analyzed. Note that, the second argument is optional and the script by default will assume that the execution time is 1440 minutes, which is equal to 1 day. For example, the command (`analyze.sh exim 240`) will analyze the first 4 hours of the execution results of the exim subject.

Upon completion of execution, the script will process the archives by construcing csv files, containing the covered number of branches, states, and state transitions over time. Furthermore, these csv files will be processed into PNG files which are plots, illustrating the average code and state coverage over time for fuzzers on each subject (`cov_over_time...`ô for the code and branch coverage,ô `state_over_time...`ô for the state and state transition coverage). All of this information is moved to aô `res_<subject name>`ô folder in the root directory with a timestamp.
