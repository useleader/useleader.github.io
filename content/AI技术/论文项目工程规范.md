# 论文项目工程规范

## 文件组织建议

推荐LaTeX项目采用以下结构：

```
your-paper/
├── template.tex      # 模板文件（格式、宏包、样式定义）
├── main.tex          # 主文件，正文全部内容写在这里
├── refs.bib          # 所有参考文献
├── imgs/             # 所有图片
├── tables/           # 所有表格
└── algs/             # 所有算法
```

### 各文件职责

- **template.tex** — 预置格式、宏包、全局设置
- **main.tex** — 所有正文内容，方便统一管理和编译
- **refs.bib** — 所有参考文献，避免散落各处

> 不建议把图片、表格、算法分别放在章节子文件夹里，统一收纳便于管理和检索。

## 版本管理建议

使用 Git 管理论文，推荐至少保持以下提交节点：

| 提交节点 | 说明 |
|---------|------|
| `chapter-outline` | 章节结构确定 |
| `figures-complete` | 图表完成 |
| `chinese-comments-complete` | 中文注释完成 |
| `english-draft` | 英文初稿 |
| `experiment-final` | 实验定稿 |
| `submission-ready` | 投稿版本 |

### 分支策略

- `main`：稳定版本
- `experiment-update`：实验更新
- `rebuttal`：回复审稿意见

### 提交类型含义

| 类型 | 含义 | 适用场景 |
|------|------|---------|
| `feat` | 新功能 | 新增章节、模块、方法 |
| `fix` | 修复 | 修正论证漏洞、修复LaTeX错误 |
| `docs` | 文档 | 写/改中文注释、润色说明 |
| `exp` | 实验 | 更新实验数据、图表 |
| `refactor` | 重构 | 结构调整、重组章节 |

### 提交信息规范

```
[类型] 简短描述

- 详细说明（可选）
```

## 文献管理

### BibTeX 规范

每条文献条目需要包含：
- 作者姓名用 `{Lastname, Firstname}` 格式（最常出错，注意检查）
- 会议/期刊名称用标准缩写（可用 [abbrevs@cc](https://ctan.org/abbrevs) 核对）
- DOI 或 URL 优先

> **来源建议**：直接从谷歌学术导出，不自己限定格式。重点检查作者姓名顺序是否正确，其他格式问题不大，编译器能跑就行。

### 术语表维护

维护一份全局术语对照表：

```
中文  |  英文
------|-------
模糊测试 | fuzzing
覆盖率 | coverage
种子生成 | seed generation
```

每次AI润色后检查术语一致性。

## 图表管理

### 图片命名

先用描述性名称，定稿后统一编号：
- `transformer-encoder.svg`（过程名）
- `fig3-transformer-encoder.svg`（终稿名）

### 表格Caption规范

```
Table 1: [主题]。实验设置：[具体设置]。统计方式：[方式]。
```

### 算法Caption规范

```
Algorithm 1: [名称]
Input: [输入]
Output: [输出]
```

## 写作工作流

```
1. 定章节结构
2. 画图梳理思路
3. 写中文注释
4. 精校中文
5. 翻译为英文
6. 精校英文
```

每阶段完成后检查：
- [ ] 逻辑通顺
- [ ] 术语一致
- [ ] 无低级错误
